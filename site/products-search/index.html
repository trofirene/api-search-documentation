Поиск товаров

Информация о запросе

Свойство Значение
Метод GET
URL /api/products/search
Тип формы query
Имя URL products_search

Данный эндпоинт позволяет выполнять поиск товаров по множеству фильтров: категориям, цене, весу, размерам, производителям, продавцам, рейтингу и характеристикам (вариантам). Поддерживает пагинацию и возвращает детальную информацию для построения фильтров на фронтенде (минимальные/максимальные значения, доступные варианты, списки производителей и продавцов).

Параметры запроса (Query Parameters)

Запрос передается в формате JSON в теле GET-запроса.

Основные фильтры

Поле Тип Описание
filters object Объект с условиями фильтрации (см. таблицу ниже).
paginate object Объект с настройками пагинации: page (номер страницы) и pp_items (элементов на странице).

Структура объекта paginate

Имя поля Описание Тип данных Обязательное Пример значения
page Номер страницы для пагинации integer Да 1
pp_items Количество элементов на странице integer Да 15

Пример запроса

{
  "filters": {
    "categories": [1, 3],
    "search_string": "брошь",
    "price_from": "1000",
    "price_to": "5000",
    "with_discount": "true",
    "ordering": "-avg_raiting",
    "variant_1": [5, 6],
    "variant_3": [9, 10]
  },
  "paginate": {
    "page": 1,
    "pp_items": 15
  }
}

Вложенные данные и фильтрация

Селекторы и дополнительные параметры

show_whishlist

Тип: Булевый селектор (string). Позволяет отфильтровать товары, находящиеся в вишлисте текущего пользователя.

Доступные значения:

  • "true" — показать только товары из вишлиста
  • "false" — показать товары не из вишлиста
  • "" (пустая строка) — не применять фильтр

only_featured

Тип: Булевый селектор (string). Фильтрация товаров, отмеченных как "рекомендуемые" (featured).

Доступные значения:

  • "true" — только рекомендуемые товары
  • "false" — только нерекомендуемые товары
  • "" — не применять фильтр

only_new

Тип: Булевый селектор (string). Фильтрация новых товаров.

Доступные значения:

  • "true" — только новые товары
  • "false" — только не новые товары
  • "" — не применять фильтр

with_discount

Тип: Булевый селектор (string). Фильтрация товаров, на которые действует скидка.

Доступные значения:

  • "true" — только товары со скидкой
  • "false" — только товары без скидки
  • "" — не применять фильтр

ordering

Тип: Селектор сортировки (string). Определяет порядок вывода результатов поиска.

Доступные значения:

  • "price" — по возрастанию цены
  • "-price" — по убыванию цены
  • "avg_raiting" — по возрастанию рейтинга
  • "-avg_raiting" — по убыванию рейтинга
  • "newest" — сначала новые
  • "popularity" — по популярности

Как использовать селекторы

Все перечисленные поля передаются внутри объекта filters в теле запроса.

Пример запроса:

{
  "filters": {
    "show_whishlist": "true",
    "only_featured": "true",
    "only_new": "false",
    "with_discount": "true",
    "ordering": "-avg_raiting"
  },
  "paginate": {
    "page": 1,
    "pp_items": 15
  }
}

Информация об ответе

В случае успешного выполнения возвращается объект с результатами поиска. Структура ответа включает агрегированные данные для построения фильтров на фронтенде: минимальные и максимальные значения цен, веса и размеров, доступные варианты характеристик, списки производителей, продавцов и категорий, а также информацию о скидках и товарах с высоким рейтингом.

Поле Описание Пример
form_errors Ошибки валидации формы. При успешном запросе — null null
success_message Сообщение об успешном выполнении "Success"
page_data Основной объект с данными ответа (см. детальную структуру ниже) {...}
exc_stack Стек ошибок (пустой при успехе) ""
debug Отладочная информация []
user_groups Группы пользователя ""
user_perms Права пользователя ""
spent Время выполнения запроса на сервере в секундах 1.5
locale Код языка, использованный для локализации ответа "ru"
url_name Техническое имя эндпоинта "products_search"

Детальная структура page_data

Поле Описание
min_max Минимальные и максимальные значения цен, веса, длины, ширины, высоты
variants Доступные варианты товаров (характеристики: цвет, размер и т.д.)
selects Списки производителей, продавцов и категорий для фильтрации
measures Детальные данные по измерениям (аналогично min_max)
discounts Информация о скидках (количество товаров со скидкой)
raiting_above_45 Количество товаров с рейтингом выше 4.5

Структура min_max / measures

Объект содержит минимальные и максимальные значения для различных метрик. Ключи (1, 2, 3...) — идентификаторы валют или единиц измерения.

Поле Описание
max_price Максимальные цены
min_price Минимальные цены
max_weight Максимальный вес
min_weight Минимальный вес
max_length Максимальная длина
min_length Минимальная длина
max_height Максимальная высота
min_height Минимальная высота
max_width Максимальная ширина
min_width Минимальная ширина

Каждое значение содержит:

  • transform_value — преобразованное значение (для отображения)
  • real_value — реальное значение (для расчетов)

Пример:

"max_price": {
  "1": { "transform_value": 26934, "real_value": 26934 }
}
